How /sharpen works

Five phases. Each one ends with a handoff, so you control the pace.
01
Capture
Echo for clear input. Structured framing for branchy input — restated goal, reusable pieces, missing pieces, numbered decisions.
adaptive
02
Probe
Settle decisions. Surface edge cases — empty, error, concurrent, permission. Confirm what's out of scope.
batched
03
Wireframeopt-in
ASCII for structure. HTML when exploring multiple directions. Claude Design prompt when refining one.
3 paths
04
Supportability
Read-only check vs the codebase. Heavy items get a forced keep / defer / cut decision before plan mode commits engineering effort.
read-only
05
Handoff
Write requirements/<slug>.md. Print transition instructions. Plan mode picks it up next.
artifact
Strict read-only on the codebase · invoke with /sharpen <ask>
